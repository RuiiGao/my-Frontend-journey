代办内容：所有标签总结

#### 一、标签
- 介绍：**HTML：超文本标记语言 —— HyperText Markup Language**
  - 超文本：即是链接
  - 标记：也叫标签，带尖括号的文本
- **标签语法** ：<开始标签> 标签操作的对象/内容 <结束标签>，例如 `<strong> 需要加粗的文字 </strong>`
  - 大多数标签**成对**出现，中间包裹内容
  - `<>` 里面放英文字母（标签名）
  - 结束标签比开始标签多了个 `/`
  - 拓展：
    - 双标签：成对出现的标签，就如加粗标签
    - 单标签：只有开始标签，没有结束标签，例如：`<br>：换行；<hr>：水平线`

#### 二、HTML基本骨架
- 介绍：HTML骨架是**网页模板**
- 形式：
  - html：**整个网页**
  - head：**网页头部**，存放给**游览器**看的代码，例如CSS
  - body：**网页主体**，存放给**用户**看的代码，例如图片、文字
  - title：**网页标题**
```html {.line-numbers}
<html>
  <head>
    <title>网页标题</title>
  </head>
  <body>
    网页主体
  </body>
</html>
```
- VS Code 中快速生成骨架：
  - 方式：在HTML文件 **（.html）** 中，输入 **!(英文)** 会出现提示，直接再按下 **`Enter / Tab`** 键即可快速生成（ `! + Enter / Tab` ）
  - 呈现的代码如下：
```html {.line-numbers}
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    输入......
</body>
</html>
```

#### 三、标签的关系
- 作用：明确代码的书写位置
- 常见关系：
  - **父子关系**（嵌套关系）：外面的大标签称为父级，里面的标签称为子级
  - **兄弟关系**（并列关系）：标签之间平级别
- 示例：
  - html、head之间互为父子关系：html为父级，head为子级
  - head、body之间互为兄弟关系
```html {.line-numbers}
<html>
  <head></head>
  <body>
    内容......
  </body>
</html>
```

#### 四、注释
- 提示：注释只会在原文档中显示，不会在游览器中显示
- 格式：`<!--注释内容-->`
- **添加 / 删除注释**的快捷键：`ctrl + /`

#### 五、标题与段落标签
- 标题标签：一般用于新闻标题、文章标题、网页区域名称、产品名称等
  - 标签名：**h1 ~ h6（双标签）**
  - 显示特点：
    - 文字加粗
    - 字号随着级数增大而逐渐减小
    - 独占一行（会换行）
  - 使用习惯：h1标签在一个网页中一般**只用一次**，用来放**新闻标题**和**网页的logo**
- 段落标签：一般用在新闻段落、文章段落、产品描述信息等
  - 标签名：**p（双标签）**
  - 显示特点：
    - 独占一行
    - 段落之间存在间隙
  - 注意：当段落文字较多时，段落中会自动换行，且会随着页面窗口大小以及视图比例的改变而自动调整换行

#### 六、换行与水平线标签
- 换行：`<br>（单标签）`
  - 注意：代码中之间用 `Enter` 来换行游览器会以一个空格的效果来呈现
- 水平线：`<hr>（单标签）`

#### 八、图像标签及其属性
- 作用：在网页中插入图片
- 整体格式：`<img src="图片的URL" alt="替换文本" title="提示文本" width="..." height="...">`
  - 注意：
    - 该标签为**单标签**
    - **src**用于指定**图像的位置和名称**，是 `<img>` 的必需属性
    - URL部分使用 `./` 可以自动出现代码提示，方便在其中选取图片
    - 属性内容按需来设置（可有可无，但src属性必须要有），如果不需要的话直接 `<img src="图片的URL"` 即可
- **属性**：
  |属性|作用|说明|
  |:-:|:-:|:-:|
  |alt|替换文本|图片无法显示的时候显示的文字|
  |title|提示文本|鼠标悬停在上面的时候显示的文字|
  |width|图片的宽度|值为数字，没有单位|
  |height|图片的高度|值为数字，没有单位|
  - 格式：属性名=“属性值”
  - 属性写在**尖括号里面**，**标签名后面**，标签名和属性之间用**空格**隔开，不区分先后顺序
  - 注意：只设置 width / height 单个属性时，游览器**会自动把图片的宽高等比缩放**

#### 九、路径
- 含义：指的是查找文件时,从**起点到终点**经历的路线
- 分类
  - 相对路径：从**当前文件位置**出发查找目标文件
    - `/` 表示**进入某个文件夹里面**（写法：`文件夹名字/` ）
    - `.` 表示**当前**文件所在文件夹（写法：`./` ）
    - `..` 表示当前文件的上一级文件夹（`../表示进入上一级文件夹里面` 那么 `../../就表示上两级文件夹` ）
  - 绝对路径：从**盘符**出发查找目标文件（Mac电脑从根目录出发）
    - Windows 默认是 `\` ，其他系统是 `/` ，建议统一写为 `/` 。
    - 文件的在线网址：如果文件不在自己电脑上，也可以复制图片链接的方式到 src 上来显示 `如：<img src="https://avatars.githubusercontent.com/u/1342004?s=48&v=4" />`
    - 绝对路径的应用场景：==友情链接==
  - 如果要找自己电脑上的文件，一般都用相对路径
